Mathematics → Extensions migration notes
========================================

Moved (now extensions)
----------------------
- MathCb.Divide(int, int, MidpointRounding)            -> Int32Extensions.Divide(this int, int, MidpointRounding)
- MathCb.Divide(long, long, MidpointRounding)          -> Int64Extensions.Divide(this long, long, MidpointRounding)
- MathCb.Divide(this uint, uint, RoundingType)         -> UInt32Extensions.Divide(this uint, uint, RoundingType)
- MathCb.Divide(this ulong, ulong, RoundingType)       -> UInt64Extensions.Divide(this ulong, ulong, RoundingType)
- MathCb.Pad(this long, int)                           -> Int64Extensions.Pad
- MathCb.Pad(this ulong, int)                          -> UInt64Extensions.Pad
- MathCb.PadLeft(double, int)                          -> DoubleExtensions.Pad (renamed Pad for naming parity with Int64/UInt64.Pad)
- MathCb.IsApproximately(this double, double, double)  -> DoubleExtensions.IsApproximately (params renamed a/b -> value/other)
- MathCb.Clamp01(double)                               -> DoubleExtensions.Clamp01
- MathCb.NonZeroSign(double)                           -> DoubleExtensions.NonZeroSign (also fixed return literal -1f -> -1d)
- MathCb.IsBetweenInclusive(double, double, double)    -> DoubleExtensions.IsBetweenInclusive
- MathCb.AreValuesMatching(int[]/long[]/uint[]/ulong[]/float[]/double[])
                                                       -> Int32/Int64/UInt32/UInt64/Single/DoubleExtensions.AreValuesMatching
- MathCb.ToInt(bool)                                   -> BooleanExtensions.ToInt (new file)

Removed (true duplicates of existing extensions)
------------------------------------------------
- MathCb.Int64ToSingleUnsafe(double value, float accuracy)
    Misnamed; signature took double and returned double. Equivalent to value * accuracy
    or DoubleExtensions.ToDouble / Int64Extensions.ToDouble. Removed.
- MathCb.DoubleToInt64Unsafe(double value, float accuracy)
    Equivalent to DoubleExtensions.ToInt64Unsafe (now also supports MidpointRounding). Removed.
- MathCb.SingleToInt32Unsafe(double value, float accuracy)
    Equivalent to SingleExtensions.ToInt32 (clamped) / ToInt32Unsafe. The original
    accepted a double which is itself suspicious. Removed.

Kept in MathCb (intentional)
----------------------------
- RandomInclusive, RandomExclusive, Random01
    Static factories. No obvious "self" to attach to; many call sites pass two operands.
- Lerp(double, double, double)
    Mirrors Mathf.Lerp / Math style; receiver would be ambiguous (start? value?).
- Vector2 / Vector3 InverseLerp and Lerp3
    Static vector math; no clear receiver.
- Clamp(float/long/ulong)
    Heavy call-site usage from many Types (IntRange/ByteRange/UIntRange/UnifiedColor/
    FloatRange2D) that lean on implicit numeric promotion. Moving these would force
    casts at every call site. Left in MathCb for now — see "Unsure / call out".
- Min(long, long), Min(ulong, ulong)
    Math.Min already covers both — these are duplicates of stdlib (see below).
- SingleToUInt32Unsafe(double, float)
    Kept but suspicious — see below.
- RandomSign(this float)
    Kept but buggy — see below.
- The shared Random field (MathCB.cs).

Unsure / call out
-----------------
1. Min(long), Min(ulong) duplicate System.Math.Min. Recommend deleting both and using
   Math.Min directly. Did not delete because they may be referenced elsewhere or be
   part of the public surface intentionally.

2. RandomSign(this float) has problems:
   - Declared on float but returns double. Caller of (float).RandomSign() loses type.
   - Uses `RandomInclusive(0UL, 1UL) == 0` — RandomInclusive returns a continuous double
     in [0, 1 + epsilon]; equality to 0 is essentially never true, so the sign is
     almost always flipped. Should be Random.Next(0, 2) == 0 with a typed Single
     overload returning float.
   - Lives in MathCB.Double.cs but is a Single extension. If you confirm intent,
     candidate to move to SingleExtensions and rewrite.

3. SingleToUInt32Unsafe(double value, float accuracy):
   - Method name says "Single" but parameter is double.
   - Equivalent to value.ToInt32Unsafe(accuracy).ToUInt32() now that the extensions
     exist. Candidate for removal once you confirm no consumer relies on the static name.

4. Clamp overloads (float/long/ulong) — strong candidates to become extensions
   (`value.Clamp(min, max)` reads naturally) but the move would touch every Range type,
   UnifiedColor, and the SingleExtensions internals, plus FloatRange2D. Held off
   because it's a wider refactor than the rest of this pass. Worth doing in a focused
   follow-up.

5. Clamp01, NonZeroSign, IsBetweenInclusive only have a `double` overload. If callers
   start needing float versions you'll want SingleExtensions equivalents to avoid the
   `(double)` casts shown in MathCb.Vector2.cs / MathCb.Vector3.cs.

6. MathCb.Vector2 / Vector3 InverseLerp call Clamp01 — they now route through
   `((double)(dotA / dotB)).Clamp01()`. Slightly noisy because no float Clamp01
   extension exists yet (see point 5).

7. CanvasGroupFader.cs IsApproximately calls were `MathCb.IsApproximately(float, float)`.
   Updated to `((double)CanvasGroup.alpha).IsApproximately(targetAlpha)`. A float
   IsApproximately overload on SingleExtensions would let this drop the cast.

8. UInt64Extensions.RoundUpToByteBitCount const is unused. Pre-existing — left alone.

9. PadLeft -> Pad rename: the previous static was MathCb.PadLeft(double, int) but the
   long/ulong helpers were already named Pad. Renamed the moved double helper to Pad
   to match. No call sites used PadLeft.
